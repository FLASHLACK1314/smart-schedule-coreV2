## 四、Dify 配置建议

### 4.1 当前配置问题

当前 DSL 中工具参数直接从 `{{#sys.query#}}` 提取，AI 需要自己从自然语言中解析参数。

### 4.2 推荐优化方案

在每个 MCP 工具前增加 LLM 参数提取节点：

```yaml
# 参数提取 LLM 节点示例
- data:
    title: 提取查询参数
    type: llm
    prompt_template:
    - role: system
      text: |
        从用户输入中提取以下参数，返回JSON格式：
        {
          "teacherName": "教师姓名",
          "dayOfWeek": 数字1-7或null,
          "semesterUuid": "学期UUID"
        }
        用户输入：{{#sys.query#}}
```

### 4.3 变量传递

```
LLM参数提取 → 工具调用 → 结果聚合 → LLM整理回复
```

## 六、快速测试命令

### 使用 MCP 工具直接测试

```sql
-- 查询张教授的排课
SELECT s.schedule_uuid, t.teacher_name, c.classroom_name,
       s.day_of_week, s.section_start, s.section_end
FROM sc_schedule s
JOIN sc_teacher t ON s.teacher_uuid = t.teacher_uuid
LEFT JOIN sc_classroom c ON s.classroom_uuid = c.classroom_uuid
WHERE t.teacher_name LIKE '%张%' AND s.status = 1;

-- 检查某时间槽冲突
SELECT * FROM sc_schedule
WHERE day_of_week = 5
  AND section_start <= 6 AND section_end >= 5
  AND status = 1;
```
