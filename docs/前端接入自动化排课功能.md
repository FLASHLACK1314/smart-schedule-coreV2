# 前端接入自动化排课功能文档

## Context

后端已实现完整的遗传算法自动排课功能，包括服务层和控制器层。前端需要对接5个API接口以实现自动排课的前端界面和交互功能。

## 接口概览

| 接口路径 | 方法 | 描述 | 权限 |
|---------|------|------|------|
| `/v1/auto-schedule/execute` | POST | 执行自动排课 | 系统管理员、教务管理员 |
| `/v1/auto-schedule/save-preview` | POST | 保存预览方案 | 系统管理员、教务管理员 |
| `/v1/auto-schedule/confirm` | POST | 确认排课方案 | 系统管理员、教务管理员 |
| `/v1/auto-schedule/clear-preview` | DELETE | 清除预览方案 | 系统管理员、教务管理员 |
| `/v1/auto-schedule/statistics` | GET | 获取排课统计信息 | 系统管理员、教务管理员 |

## 接口详情

### 1. 执行自动排课

**接口路径**: `POST /v1/auto-schedule/execute`

**请求头**:
```
Authorization: Bearer {token}
```

**请求体** (AutoScheduleVO):
```json
{
  "semester_uuid": "string",

  "teaching_class_uuids": ["string"],
  "course_class_mapping": {"string": ["string"]},

  "teacher_assignment": {"string": "string"},
  "teacher_selection_strategy": "balanced",

  "building_uuids": ["string"],
  "classroom_type_uuids": ["string"],

  "weekly_sessions_config": {"string": 0},
  "sections_per_session_config": {"string": 0},

  "schedule_mode": 0,
  "overwrite": false,

  "population_size": 100,
  "max_generations": 500,
  "crossover_rate": 0.8,
  "mutation_rate": 0.2,
  "elite_size": 10
}
```

**两种排课模式说明**：

#### 模式一：按教学班排课（传统模式）
- 适用于已有教学班的情况
- 需要先创建教学班，然后选择要排课的教学班
- 参数：提供 `teaching_class_uuids`

#### 模式二：按行政班级排课（新模式，推荐）
- 适用于直接从课程和行政班角度进行排课
- 前端只需提供"课程UUID → 行政班列表"的映射
- 后端自动：
  1. 智能匹配或创建教学班
  2. 从 `sc_course_qualification` 表查询教师资格
  3. 自动分配教师
  4. 保证合班上课时同一课程的多个行政班在同一时间
- 参数：提供 `course_class_mapping`
- 支持合班上课：同一课程可关联多个行政班

**响应**:

## 相关接口

除了5个自动排课核心接口外，还需要以下接口支持：

- `GET /v1/semester/getPage` - 获取学期列表
- `GET /v1/teaching-class/getPage` - 获取教学班列表（传统模式）
- `GET /v1/course/getPage` - 获取课程列表（新模式）
- `GET /v1/class/getPage` - 获取行政班列表（新模式）
- `GET /v1/course-qualification/teachers` - 获取课程的教师列表（可选，用于展示）

## 请求参数说明

### AutoScheduleVO 参数

| 参数名 | 类型 | 必填 | 说明 | 默认值 |
|--------|------|------|------|--------|
| semester_uuid | string | 是 | 学期UUID | - |
| teaching_class_uuids | string[] | 否* | 旧模式：待排课的教学班UUID列表 | - |
| course_class_mapping | object | 否* | 新模式：课程-行政班映射 | - |
| teacher_assignment | object | 否 | 指定教师映射（课程UUID -> 教师UUID） | - |
| teacher_selection_strategy | string | 否 | 教师选择策略：balanced/random/first | balanced |
| building_uuids | string[] | 否 | 可用教学楼UUID列表 | 查询所有 |
| classroom_type_uuids | string[] | 否 | 可用教室类型UUID列表 | 使用所有类型 |
| weekly_sessions_config | object | 否 | 每周上课次数配置 | 使用教学班配置 |
| sections_per_session_config | object | 否 | 每次上课节次数配置 | 使用教学班配置 |
| schedule_mode | number | 否 | 排课模式：0-预览, 1-正式 | 0 |
| overwrite | boolean | 否 | 是否覆盖已有排课记录 | false |
| population_size | number | 否 | 种群大小 | 100 |
| max_generations | number | 否 | 最大迭代代数 | 500 |
| crossover_rate | number | 否 | 交叉概率 | 0.8 |
| mutation_rate | number | 否 | 变异概率 | 0.2 |
| elite_size | number | 否 | 精英保留数量 | 10 |

**注意**：`teaching_class_uuids` 和 `course_class_mapping` 必须提供其中之一

## 排课流程

### 标准排课流程（传统模式）

```
用户选择学期
  ↓
加载该学期的教学班列表
  ↓
用户选择要排课的教学班（支持多选）
  ↓
（可选）配置资源范围
  ↓
（可选）配置算法参数
  ↓
选择排课模式（预览/正式）
  ↓
点击"开始排课"
  ↓
显示进度和日志
  ↓
算法完成，展示结果
  ↓
查看课表、冲突报告、统计信息
  ↓
根据结果选择操作：
  - 保存预览 → 保存为预览状态
  - 确认方案 → 将预览转为正式方案
  - 重新排课 → 调整参数后重试
```

### 按行政班级排课流程（新模式，推荐）

```
用户选择学期
  ↓
加载课程列表
  ↓
用户选择课程
  ↓
加载该课程的行政班列表
  ↓
用户选择要排课的行政班（支持多选，用于合班上课）
  ↓
（可选）为课程指定教师或选择教师分配策略
  ↓
添加到课程-行政班映射中
  ↓
重复以上步骤添加更多课程
  ↓
（可选）配置资源范围
  ↓
（可选）配置算法参数
  ↓
选择排课模式（预览/正式）
  ↓
点击"开始排课"
  ↓
后端自动处理：匹配/创建教学班、分配教师、确保合班时间一致
  ↓
显示进度和日志
  ↓
算法完成，展示结果
  ↓
查看课表、冲突报告、统计信息
  ↓
根据结果选择操作：
  - 保存预览 → 保存为预览状态
  - 确认方案 → 将预览转为正式方案
  - 重新排课 → 调整参数后重试
```

### 预览模式流程

```
执行排课（schedule_mode=0）
  ↓
查看排课结果
  ↓
如满意 → 保存预览
  ↓
继续调整其他教学班
  ↓
所有排课满意 → 确认方案
  ↓
所有预览转为正式方案
```

### 正式模式流程

```
执行排课（schedule_mode=1）
  ↓
排课结果直接写入正式课表
  ↓
查看课表和统计
  ↓
如需调整：
  - 清除已有排课（overwrite=true）
  - 重新执行排课
```

## 新模式优势

按行政班级排课模式相比传统模式有以下优势：

1. **简化流程**：无需提前创建教学班，直接从课程和行政班角度操作
2. **智能匹配**：自动复用已有教学班或创建新教学班
3. **自动分配**：自动选择有资格的教师，减少手动工作
4. **合班支持**：天然支持合班上课，确保同一课程的多个行政班在同一时间
5. **灵活配置**：支持指定特定教师或采用不同策略自动选择
6. **向后兼容**：保留传统模式，现有系统无需改造
