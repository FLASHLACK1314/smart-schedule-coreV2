# 前端对接文档 - 数据库字段更新

## 更新概述

本次更新为教学班（Teaching Class）新增了两个字段，用于支持智能排课功能：

- **weeklySessions**: 每周上课次数
- **sectionsPerSession**: 每次上课节次数

---

## 数据库变更

### 表结构变更

**表名**: `sc_teaching_class`

**新增字段**:

| 字段名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| weekly_sessions | INTEGER | 1 | 每周上课次数 |
| sections_per_session | INTEGER | 2 | 每次上课节次数 |

**约束条件**:
- `weekly_sessions > 0`
- `sections_per_session > 0`

---

## API 变更

### 1. 添加教学班

**接口**: `POST /teaching-class/add`

#### 旧版本请求

```json
{
  "courseUuid": "course123",
  "teacherUuid": "teacher456",
  "semesterUuid": "semester789",
  "teachingClassName": "高等数学-计科2101班"
}
```

#### 新版本请求

```json
{
  "courseUuid": "course123",
  "teacherUuid": "teacher456",
  "semesterUuid": "semester789",
  "teachingClassName": "高等数学-计科2101班",
  "weeklySessions": 2,          // 新增：每周上课次数（可选）
  "sectionsPerSession": 2       // 新增：每次上课节次数（可选）
}
```

**向后兼容性**: 这两个新字段是可选的，如果不传则使用默认值（weeklySessions=1, sectionsPerSession=2）

### 4. 更新 API 请求

**文件**: `src/api/teaching-class.ts`

```typescript
import request from '@/utils/request';
import type { AddTeachingClassForm, TeachingClass } from '@/types/teaching-class';

/**
 * 添加教学班
 */
export const addTeachingClass = (data: AddTeachingClassForm) => {
  return request.post<{ data: string }>('/teaching-class/add', {
    ...data,
    // 确保新字段被包含在请求中
    weeklySessions: data.weeklySessions ?? 1,
    sectionsPerSession: data.sectionsPerSession ?? 2
  });
};

/**
 * 更新教学班
 */
export const updateTeachingClass = (data: AddTeachingClassForm) => {
  return request.post('/teaching-class/update', {
    ...data,
    weeklySessions: data.weeklySessions ?? 1,
    sectionsPerSession: data.sectionsPerSession ?? 2
  });
};

/**
 * 查询教学班信息
 */
export const getTeachingClass = (uuid: string) => {
  return request.get<{ data: TeachingClass }>(`/teaching-class/${uuid}`);
};
```

## 常见问题

### Q1: 旧数据会有这两个字段吗？

**A**: 数据库默认值为 `weekly_sessions=1`, `sections_per_session=2`，旧数据会自动填充默认值。

### Q2: 这两个字段是必填的吗？

**A**: 不是必填的。如果不传，后端会使用默认值。

### Q3: 前端需要立即更新吗？

**A**: 不需要立即更新。API 是向后兼容的，但建议尽快更新以支持智能排课功能。

### Q4: 智能排课如何使用这两个字段？

**A**:
- `weeklySessions`: 控制每周排几次课
- `sectionsPerSession`: 控制每次排几节课（目前建议固定为2）

例如：32学时的课程，如果 `weeklySessions=2`，则需要 8 周完成（每周2次 × 2节 × 8周 = 32学时）。
