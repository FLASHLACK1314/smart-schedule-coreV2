# 智能排课功能前端对接文档

## 功能概述

智能排课系统基于遗传算法，根据教学班、教师、教室等约束条件自动生成排课方案。

### 核心特性

- **自动排课**: 基于遗传算法自动优化排课方案
- **约束检测**: 自动检测教师、教室、班级时间冲突
- **预览确认**: 支持预览排课结果，确认后再生效
- **灵活配置**: 支持自定义算法参数和每周上课次数

---

## API 接口总览

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 执行自动排课 | POST | `/auto-schedule/execute` | 执行排课算法，返回预览结果 |
| 确认排课方案 | POST | `/auto-schedule/confirm` | 将预览状态转为正式状态 |
| 清除预览排课 | POST | `/auto-schedule/clear` | 清除预览状态的排课记录 |
| 查询排课结果 | GET | `/auto-schedule/result/{semesterUuid}` | 查询学期的排课结果 |

## 核心接口详解

### 1. 执行自动排课

**接口路径**: `POST /auto-schedule/execute`

**请求参数**:

```typescript
interface AutoScheduleRequest {
  semesterUuid: string;                           // 学期UUID（必填）
  teachingClassUuids: string[];                   // 待排课的教学班UUID列表（必填）
  weeklySessionsConfig?: Record<string, number>;  // 每周上课次数配置（可选）
  classroomUuids?: string[];                      // 可用教室UUID列表（可选）
  populationSize?: number;                        // 种群大小（可选，默认100）
  maxGenerations?: number;                        // 最大迭代次数（可选，默认500）
  crossoverRate?: number;                         // 交叉概率（可选，默认0.8）
  mutationRate?: number;                          // 变异概率（可选，默认0.2）
  eliteSize?: number;                             // 精英保留数量（可选，默认10）
}
```

**响应数据**:

```typescript
interface ScheduleResult {
  scheduleId: string;                              // 排课方案ID
  status: 'preview' | 'confirmed';                 // 方案状态
  statistics: {
    totalTeachingClasses: number;                  // 教学班总数
    scheduledTeachingClasses: number;              // 已排课教学班数
    totalSchedules: number;                        // 排课记录总数
    hardConstraintViolations: number;              // 硬约束违反次数
    softConstraintViolations: number;              // 软约束违反次数
  };
  schedules: ScheduleItem[];                       // 排课记录列表
  conflicts: ConflictItem[];                       // 冲突记录列表
}

interface ScheduleItem {
  scheduleUuid: string;          // 排课记录UUID
  teachingClassUuid: string;     // 教学班UUID
  teachingClassName: string;     // 教学班名称
  courseUuid: string;            // 课程UUID
  courseName: string;            // 课程名称
  teacherUuid: string;           // 教师UUID
  teacherName: string;           // 教师名称
  classroomUuid: string;         // 教室UUID
  classroomName: string;         // 教室名称
  dayOfWeek: number;             // 星期几（1-7）
  sectionStart: number;          // 起始节次
  sectionEnd: number;            // 结束节次
  weeks: number[];               // 上课周次列表
  creditHours: number;           // 本次排课学时
}

interface ConflictItem {
  conflictUuid: string;          // 冲突记录UUID
  conflictType: string;          // 冲突类型
  description: string;           // 冲突描述
  relatedSchedules: string[];    // 相关排课记录UUID
}
```

## 前端实现示例

### 1. API 封装

**文件**: `src/api/auto-schedule.ts`

```typescript
import request from '@/utils/request';
import type {
  AutoScheduleRequest,
  ScheduleResult,
  ConfirmScheduleRequest,
  ClearPreviewRequest,
  ScheduleQueryResult
} from '@/types/auto-schedule';

/**
 * 执行自动排课
 */
export const executeAutoSchedule = (data: AutoScheduleRequest) => {
  return request.post<{ data: ScheduleResult }>('/auto-schedule/execute', data);
};

/**
 * 确认排课方案
 */
export const confirmSchedule = (data: ConfirmScheduleRequest) => {
  return request.post<{ data: { updatedCount: number } }>('/auto-schedule/confirm', data);
};

/**
 * 清除预览排课
 */
export const clearPreviewSchedule = (data: ClearPreviewRequest) => {
  return request.post<{ data: { deletedCount: number } }>('/auto-schedule/clear', data);
};

/**
 * 查询排课结果
 */
export const getScheduleResult = (semesterUuid: string) => {
  return request.get<{ data: ScheduleQueryResult }>(`/auto-schedule/result/${semesterUuid}`);
};
```

<script setup lang="ts">
import { ref, reactive, computed } from 'vue';
import { ElMessage, ElMessageBox } from 'element-plus';
import { executeAutoSchedule, confirmSchedule, clearPreviewSchedule } from '@/api/auto-schedule';
import type { AutoScheduleRequest, ScheduleResult } from '@/types/auto-schedule';

const step = ref(1);
const loading = ref(false);
const showAdvanced = ref(false);
const result = ref<ScheduleResult>({} as ScheduleResult);

const form = reactive<AutoScheduleRequest>({
  semesterUuid: '',
  teachingClassUuids: [],
  populationSize: 100,
  maxGenerations: 500,
  eliteSize: 10
});

const weeklyConfig = ref<Record<string, number>>({});

// 计算完成率
const completionRate = computed(() => {
  const stats = result.value.statistics;
  if (!stats) return 0;
  return Math.round((stats.scheduledTeachingClasses / stats.totalTeachingClasses) * 100);
});

// 是否可以确认（无硬冲突）
const canConfirm = computed(() => {
  const stats = result.value.statistics;
  return stats && stats.hardConstraintViolations === 0;
});

// 执行排课
const handleExecute = async () => {
  loading.value = true;
  try {
    const res = await executeAutoSchedule({
      ...form,
      weeklySessionsConfig: weeklyConfig.value
    });

    result.value = res.data;
    step.value = 2;

    ElMessage.success(`排课完成，共生成 ${res.data.schedules.length} 条排课记录`);
  } catch (error) {
    ElMessage.error('排课失败: ' + error.message);
  } finally {
    loading.value = false;
  }
};

// 确认方案
const handleConfirm = async () => {
  try {
    await ElMessageBox.confirm('确认后将生成正式排课记录，是否继续？', '确认操作');

    await confirmSchedule({ semesterUuid: form.semesterUuid });
    ElMessage.success('排课方案已确认');
  } catch (error) {
    if (error !== 'cancel') {
      ElMessage.error('确认失败: ' + error.message);
    }
  }
};

// 清除预览
const handleClear = async () => {
  try {
    await ElMessageBox.confirm('确定要清除当前预览方案吗？', '提示');

    await clearPreviewSchedule({ semesterUuid: form.semesterUuid });
    ElMessage.success('预览方案已清除');
    handleReset();
  } catch (error) {
    if (error !== 'cancel') {
      ElMessage.error('清除失败: ' + error.message);
    }
  }
};

// 重新排课
const handleReset = () => {
  step.value = 1;
  result.value = {} as ScheduleResult;
};

// 格式化时间
const formatTime = (row: any) => {
  const weekMap = ['', '周一', '周二', '周三', '周四', '周五', '周六', '周日'];
  return `${weekMap[row.dayOfWeek]} ${row.sectionStart}-${row.sectionEnd}节`;
};

// 格式化周次
const formatWeeks = (weeks: number[]) => {
  if (weeks.length === 0) return '';
  if (weeks.length <= 8) {
    return weeks.join('、');
  }
  return `${weeks[0]}-${weeks[weeks.length - 1]}周`;
};
</script>

## 业务流程说明

### 完整排课流程

```
1. 用户选择学期和教学班
   ↓
2. 配置每周上课次数（可选）
   ↓
3. 点击"开始排课"按钮
   ↓
4. 后端执行遗传算法，生成预览方案
   ↓
5. 前端展示排课结果预览
   ↓
6. 用户查看结果
   ├─ 满意 → 点击"确认方案" → 方案生效（status: 0→1）
   └─ 不满意 → 点击"清除预览" → 重新排课
```

### 状态说明

| 状态 | 值 | 说明 |
|------|-----|------|
| 预览 | 0 | 排课算法生成的临时方案，可删除 |
| 正式 | 1 | 用户确认后的正式排课，不可通过清除接口删除 |

## 常见问题

### Q1: 排课需要多长时间？

**A**: 取决于教学班数量和算法参数配置：
- 小规模（< 50个教学班）：约 1-2 分钟
- 中规模（50-150个教学班）：约 3-5 分钟
- 大规模（> 150个教学班）：约 5-10 分钟

### Q2: 如何理解每周上课次数配置？

**A**:
- `weeklySessions=1`: 每周上1次课，例如：周一1-2节
- `weeklySessions=2`: 每周上2次课，例如：周一1-2节、周三3-4节
- `weeklySessions=3`: 每周上3次课，例如：周一、周三、周五各一次

### Q3: 硬约束和软约束有什么区别？

**A**:
- **硬约束**: 必须满足的约束（如教师不能同时上两门课）
- **软约束**: 优化目标（如教师时间偏好）

硬约束违反数为0才能确认方案。

### Q4: 预览方案会覆盖正式排课吗？

**A**: 不会。预览方案（status=0）和正式排课（status=1）是独立的，清除预览不会影响正式排课。

### Q5: 如何处理排课失败？

**A**: 如果完成率较低或硬约束过多，可以尝试：
1. 减少同时排课的教学班数量
2. 增加可用教室资源
3. 调整每周上课次数配置
4. 增加算法迭代次数
